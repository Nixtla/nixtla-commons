To contribute to Nixtla's development, it's necessary to install it first.

There are a couple of different installation methods you can select from:

- from [source](/setup/self-hosted/pip/source/),
- using PyPi for [Windows](/setup/self-hosted/pip/windows/), [Linux](/setup/self-hosted/pip/linux/), or [MacOS](/setup/self-hosted/pip/macos/),
- using a [Docker image](/setup/self-hosted/docker/).

The most recommended method of installing Nixtla for development is from the source.

## Installing Nixtla

In this section, we revisit the steps to install Nixtla from source. You can follow the steps outlined here or refer to the provided link.

Before beginning, ensure you have `Git` and either `Python 3.8.x` or `Python 3.9.x` installed.

1. Fork the Nixtla repository on
   [GitHub](https://github.com/nixtla/nixtla/fork).

2. Clone the forked repository locally:

```bash
git clone git@github.com:YOUR_USERNAME/nixtla.git
```

3. Create a new virtual environment:

```bash
python -m venv nixtla-venv
```

4. Activate the virtual environment:

```bash
source nixtla-venv/bin/activate
```

5. Install the dependencies:

```bash
cd nixtla && pip install -r requirements.txt
```

6. Install Nixtla:

```bash
python setup.py develop
```

7. Start Nixtla:

```bash
python -m nixtla
```

## What's Next?

**Community Tutorials** sections, where various examples of regression, classification, and time series predictions using Nixtla are available.
